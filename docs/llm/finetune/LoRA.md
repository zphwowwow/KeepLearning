# 解释一下 LoRA 的原理和优势
LoRA 的全称是低秩适配。它的核心思想是：预训练大模型的权重矩阵通常是满秩的，但微调时更新的部分其实是低秩的。LoRA 不直接去更新原来的权重矩阵 W，而是给 W 旁边加一个旁路，这个旁路由两个小矩阵 A 和 B 相乘得到，其中 A 是 d×r，B 是 r×k，r 远小于 d 和 k。微调时只更新 A 和 B，原来的 W 冻结不动。前向计算时，输出就是 Wx 加上 BAx。

它的优势很明显：第一，参数量大大减少，比如 r=8 时，可训练参数只有原来的万分之一左右；第二，显存占用低，因为不需要存储整个模型的梯度；第三，可以快速切换任务，只要保存不同的 LoRA 权重插回去就行，不需要复制整个模型。

面试时如果能补充一句：LoRA 本质假设是微调时的参数变化是一个低秩矩阵，这个假设在很多任务上被验证是成立的。

# **LoRA 微调的数学原理是什么？它的 rank 如何选择？QLoRA 在 LoRA 基础上做了哪些改进？**

**LoRA 的数学原理**：  
它假设微调时权重的变化是低秩的。原来权重矩阵 W 是 d×k，LoRA 不直接更新 W，而是加一个旁路：W' = W + BA。B 是 d×r，A 是 r×k，r 远小于 d 和 k。训练时只更新 A 和 B，W 冻结。前向就是 Wx + BAx，这样参数量从 d×k 降到 r×(d+k)，非常省显存。

**Rank 怎么选**：  
r 一般在 4 到 64 之间。简单任务用 4 或 8 就够了，复杂任务可以到 16 或 32。如果数据量大且希望全参数微调的效果，可以尝试 64。实际可以从小 r 开始，看验证集效果，够用就行，不需要追求大 r。

**QLoRA 的改进**：  
QLoRA 在 LoRA 基础上加了量化。它把基础模型 W 量化到 4-bit（通常是 NF4 格式），大幅减少显存。然后 LoRA 的 A 和 B 仍然用 FP16 训练。另外引入双重量化和分页优化器，进一步省显存。效果是：原本 7B 模型全参数微调要 24G+ 显存，LoRA 要 16G 左右，QLoRA 只要 6-8G，而且精度损失很小。
