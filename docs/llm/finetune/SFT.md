 # **SFT（监督微调）时数据格式如何构造？Prompt模板重要吗？**
 监督微调的数据格式通常是这样的：一条样本包含 instruction、input 和 output 三个字段。训练时我们把 instruction 和 input 拼成类似“[INST] 请解释什么是机器学习 [/INST]”这样的格式，后面跟上模型的回答，然后在回答部分计算 loss，指令部分是要 mask 掉的。

Prompt 模板极其重要，因为训练和推理必须用完全一样的模板。常见的模板有 ChatML、Alpaca 和 ShareGPT 三种。如果训练时用的是一种模板，推理时换了另一种，模型很可能不知道什么时候该停止生成，或者对话格式就乱掉了。

#  **SFT（监督微调）和 RLHF 有什么区别？为什么通常先做 SFT 再做 RLHF？**

SFT 和 RLHF 的区别，简单说就是：**SFT 是教模型“该怎么回答”，RLHF 是教模型“什么回答更好”**。

具体来说，SFT 用的是人工标注的“问题-正确答案”对，训练目标是让模型输出尽可能接近标准答案，属于监督学习。RLHF 用的是人类偏好数据，比如给你两个回答让你选哪个更好，然后训练一个奖励模型来模拟人类的偏好，再用强化学习（通常是 PPO）去优化模型，让模型生成的回答能获得更高的奖励。

为什么通常先做 SFT 再做 RLHF？因为 SFT 是基础。如果不做 SFT，模型连基本的对话格式和指令跟随能力都没有，生成的内容可能乱七八糟，这时候直接做 RLHF 效率极低，甚至根本训不出来。SFT 先给模型一个“好行为”的起点，让它知道该怎么说话、怎么回答问题。然后在 RLHF 阶段，再在这个基础上微调，让模型学会更符合人类偏好，比如更安全、更 helpful、更诚实。如果跳过 SFT，RLHF 就像是让一个不会走路的人去学跑步，基本不可能。

所以标准的三阶段流程就是：预训练 -> SFT -> RLHF。SFT 是让模型学会指令跟随，RLHF 是对齐人类偏好。

# **你在微调一个对话模型时，发现模型产生幻觉（hallucination）严重，你会从数据、训练、推理三个层面分别采取什么措施？**

**数据层面**：第一，清洗训练数据，删掉那些包含事实错误或者回答与问题不匹配的样本。第二，增加高质量、可验证的数据，比如带引用的问答对，让模型学会“不知道就说不知道”。第三，做负样本挖掘，把那些容易产生幻觉的问题和错误回答作为反面例子加进去。

**训练层面**：第一，用 contrastive learning，让模型对正确回答和幻觉回答的区分更明显。第二，加一个约束 loss，比如鼓励模型在不确定时输出“我不确定”而不是瞎编。第三，用 RLHF 时，把事实准确性作为奖励模型的一个重要维度，给准确回答高分，给幻觉回答扣分。

**推理层面**：第一，调整解码参数，降低 temperature（比如 0.1-0.3），减少随机性。第二，使用检索增强生成（RAG），先搜相关文档再生成，让答案有依据。第三，做自一致性采样，多次生成取多数答案，或者让模型自己验证输出是否合理。第四，加一个事实检查模块，对关键事实进行二次验证。
