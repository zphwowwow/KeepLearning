# **推导一下Attention的公式，并说明它为什么是O(n²)的？**

Attention 的核心公式是 softmax(QK^T / √d_k) 再乘以 V。这里的 Q、K、V 都是输入 X 乘上权重矩阵得到的。复杂度主要来自 QK^T 这一步矩阵乘法：序列长度是 n，Q 和 K 的每一行做点积，得到 n×n 的注意力矩阵，所以是 O(n²) 的。

除以 √d_k 是为了防止点积的值太大。因为 Q 和 K 的每个元素通常是均值为 0、方差为 1 的，两个向量的点积方差就等于 d_k，维度越大点积的数值就越极端。太大的值进 softmax 之后，最大的那个 logit 会压倒所有其他值，softmax 的输出几乎变成 one‑hot，梯度就会趋近于 0，模型就学不动了。除以 √d_k 可以把方差拉回 1，让 softmax 的梯度保持正常。

# Transformer 中 Self-Attention 的计算复杂度是多少？如果序列长度是 n，嵌入维度是 d，请分析原因。有什么方法降低长序列的注意力计算开销？

**复杂度是多少？为什么？**

Self-Attention 的复杂度是 **O(n² × d)**，其中 n 是序列长度，d 是嵌入维度。为什么是 O(n² × d) 呢？我们可以拆成三步看：

第一步，计算 Q、K、V：输入 X 是 n×d，分别乘上三个 d×d 的权重矩阵，这一步是 O(n × d²)，如果 d 不是特别大，这部分不是瓶颈。

第二步，计算注意力分数矩阵：Q 和 K 的转置相乘，Q 是 n×d，K^T 是 d×n，得到 n×n 的矩阵。这个矩阵乘法要做 n² 次点积，每次点积是 d 维向量的内积，所以复杂度是 O(n² × d)。这是最主要的部分。

第三步，注意力矩阵乘以 V：n×n 的矩阵乘 n×d 的矩阵，复杂度也是 O(n² × d)。

所以整体就是 O(n² × d)。当 n 很大时（比如几万甚至几十万），n² 项会爆炸，这是长序列处理的瓶颈。

**怎么降低长序列的注意力开销？**

有几种主流思路：

第一，**稀疏注意力**。不让每个 token 看到所有历史 token，只让它看一部分。比如 **滑动窗口注意力**，每个 token 只看前后固定窗口内的 token，复杂度降到 O(n × window)。Longformer 和 BigBird 都用了这种思路，再加上一些全局 token（比如 [CLS]）能看到全部，保证信息流通。

第二，**线性注意力**。通过改变计算顺序，把 (QK^T)V 变成 Q(K^TV)，利用矩阵乘法的结合律。本来 QK^T 是 n×n，但先算 K^TV 是 d×d，再乘 Q 变成 n×d，复杂度从 O(n² d) 降到 O(n d²)。如果 d 远小于 n，效果很明显。典型代表是 Linear Transformer 和 Performer（用随机特征近似 softmax）。

第三，**低秩近似**。认为注意力矩阵是低秩的，可以用低秩矩阵去逼近。例如 Reformer 用了局部敏感哈希（LSH），把相似的 token 分到同一个桶里，只在桶内做 attention。还有 Nyström 方法，用一小部分 landmark 来近似整个注意力矩阵。

第四，**分块与重计算**，比如 **FlashAttention**。虽然它没有改变 O(n²) 的复杂度，但通过分块计算和避免把完整 n×n 矩阵写到显存，大幅减少了显存访问次数，实际速度能快 2-4 倍，而且可以处理更长的序列。

第五，**结构化的状态空间模型**，比如 Mamba。它干脆不用 attention，用 SSM 达到线性复杂度，这是另一条技术路线，最近也很火。

总结一下，实际工程中常用组合方案：比如用滑动窗口 + 全局 token 把复杂度降到 O(n)，或者用 FlashAttention 硬扛到 128k 长度，再长就考虑线性注意力或 Mamba。

# 什么是稀疏注意力？
稀疏注意力简单说就是不让每个 token 和所有 token 都做交互，而是只选一小部分来做。常用的模式有三种：滑动窗口，每个 token 只看前后固定范围内的邻居；全局注意力，选几个全局 token 让它们能看到全部、全部也能看到它们；还有基于内容的，比如用哈希把相似的 token 分到同一组，组内做 attention。复杂度从 O(n²) 降到 O(n × 窗口大小)。优点是能处理更长的序列，显存和计算都省很多。缺点是需要设计合理的稀疏模式，实现比标准 attention 复杂，而且表达能力理论上会比全 attention 弱一点，但实际中滑动窗口加全局 token 已经够用。

# 为什么 LLM 通常使用 LayerNorm 而不是 BatchNorm？在 Pre-Norm 和 Post-Norm 结构之间你如何选择？

**为什么用 LayerNorm 而不是 BatchNorm？**

核心原因是：BatchNorm 依赖 batch 维度做归一化，它假设同一个 batch 里不同样本的同一位置特征分布是稳定的。但 LLM 训练时有两个问题：第一，序列长度变化很大，不同样本长度不一样，padding 会干扰统计量；第二，自回归生成时，每次只能处理当前 token，没有完整的 batch 统计可用。LayerNorm 是在特征维度上做归一化，跟 batch 大小和序列长度都没关系，每个 token 独立计算自己的均值和方差，天然适合变长序列和自回归生成。另外，LayerNorm 不需要维护 running mean 和 variance，训练和推理行为一致，更简单稳定。

**Pre-Norm 和 Post-Norm 怎么选？**

Post-Norm 是原始 Transformer 的结构：先做 attention 或 FFN，然后加残差，最后做 LayerNorm。Pre-Norm 是先把输入过 LayerNorm，再做 attention 或 FFN，再加残差。现在的 LLM 基本都选 Pre-Norm。

原因是：Post-Norm 在深层容易出现梯度消失，尤其是层数很深时，残差路径上的梯度会被 LayerNorm 压缩，需要做 warmup 或特殊的初始化才能稳定训练。Pre-Norm 的梯度路径更顺畅，每层的输入都经过归一化后进入子层，残差连接直接传递未经缩放的梯度，训练非常稳定，不需要 warmup 就能收敛。虽然 Pre-Norm 在极深时可能会让靠近输出的层对输入的影响变小（有论文说它表示能力稍弱），但实际中 Pre-Norm 的稳定性和训练效率优势远大于这点理论损失。所以现在 GPT、LLaMA、Mistral 全都在用 Pre-Norm。
