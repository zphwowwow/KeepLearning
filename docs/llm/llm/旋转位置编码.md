# 旋转位置编码（RoPE）相比绝对位置编码有什么优势？
旋转位置编码 RoPE 的核心想法是：把位置信息直接旋转到 Query 和 Key 的向量上。最后计算 attention 分数的时候，点积的结果天然只依赖于两个 token 的相对位置差，而不是绝对位置。

它的好处有三个：第一，因为有相对位置信息，模型对长序列的外推能力更好，比如用 2048 的序列训练的模型，可以泛化到 4096 甚至更长。第二，它不需要额外增加参数量，旋转是一个确定的数学运算。第三，它不改变 attention 的内积形式，所以可以和线性 attention 等方法兼容。

现在像 LLaMA、PaLM 这些主流模型都在用 RoPE。

# RoPE（旋转位置编码）是如何实现相对位置信息的？相比绝对位置编码和 ALiBi，它的优缺点是什么？
RoPE 实现相对位置信息的方式很巧妙。它不是在输入上加位置向量，而是把位置信息直接旋转到 Query 和 Key 的向量上。具体来说，对于位置 m 的 token，先把它的 Query 向量乘上一个旋转矩阵；对于位置 n 的 Key，也乘上对应的旋转矩阵。两个旋转后的向量做点积，结果天然只依赖于 m 和 n 的差值，而不依赖于它们各自的绝对位置。数学上相当于在复数空间给每个维度乘上不同频率的旋转因子，最终点积里出现的是 m-n 的正余弦组合。

对比绝对位置编码，绝对位置编码直接加一个可学习的或固定的位置向量，缺点是只能记住绝对位置，外推性差——训练 2048，推理 4096 时模型就懵了。RoPE 不需要额外参数，且外推能力强，因为它天生就是相对位置。

对比 ALiBi，ALiBi 不改变 attention 计算本身，而是给注意力分数加一个跟距离成正比的惩罚偏置，距离越远惩罚越大。ALiBi 的好处是简单高效，外推性极好，甚至可以不用微调直接外推很多倍。缺点是这个偏置是固定的，不像 RoPE 那样融入向量空间，可能在需要精细的相对位置建模的任务上稍弱。RoPE 和 ALiBi 都是目前主流，LLaMA、PaLM 用 RoPE，BLOOM、一些早期模型用 ALiBi。我个人认为 RoPE 更优雅，因为它把位置信息融入了特征表示，而 ALiBi 更像一个后处理的启发式规则。不过 ALiBi 在极长文本外推（比如 2k 训到 20k 推理）上表现非常惊艳，RoPE 通常需要配合插值方法才能达到类似效果。
