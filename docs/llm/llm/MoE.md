# 什么是 MoE？有什么优缺点？
MoE 是混合专家模型。它的结构是：把传统 FFN 层换成多个并行的“专家”网络，外加一个门控网络。每个 token 进来时，门控网络会选其中几个专家（比如 top-2）来计算，然后把它们的输出加权求和。

优点主要有两个：一是可以在不显著增加计算量的前提下大幅增加参数量，因为每个 token 只激活少数专家，推理速度基本不变。比如 Mixtral 8x7B 总参数量 47B，但每次推理只激活 13B 左右。二是不同的专家可以自动学习到不同的模式，比如有的专家擅长数学，有的擅长代码。

缺点也很明显：第一，训练不稳定，门控网络容易崩溃，导致所有 token 都跑去同一个专家，需要加负载平衡损失。第二，推理时显存占用大，因为虽然计算量没涨，但所有专家的参数都要加载到显存里。第三，微调时也容易出问题，比如某些专家被过度使用。

# **你认为 MoE（混合专家模型）相比稠密模型有什么优势和劣势？在训练和推理时需要注意哪些工程问题？**

好的，直接说。

**优势**：MoE 最大的好处是：参数量大但计算量小。比如 Mixtral 8x7B，总参数量 47B，但每次推理只激活 13B，速度和 13B 差不多，效果却接近 40B 级别。它通过门控网络每次只挑 top‑2 专家算，其他专家闲着，相当于用“稀疏激活”换来了大参数量。

**劣势**：第一，训练不稳定，门控容易坍塌，所有 token 都跑去同一个专家，需要加负载平衡损失。第二，显存占用大，虽然计算量小，但所有专家的参数都得加载到显存里，推理时对显存压力大。第三，微调容易翻车，有些专家被过度使用，有些闲置，需要小心调参。

**工程注意点**：训练时一定要加负载平衡损失，让每个专家的使用率差不多。通信开销也大，专家分布在多卡上，token 需要做 all‑to‑all 通信，网络带宽容易成为瓶颈。推理时可以用 expert parallelism，把不同专家放到不同 GPU 上，但调度复杂。另外，MoE 对 batch size 敏感，batch 太小门控可能选不准，batch 大才能发挥优势。
