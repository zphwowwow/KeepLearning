# 什么是 RAG？和微调相比有什么优劣？
RAG 是检索增强生成。它的流程是：先根据用户的 query 从一个知识库中检索出相关的文档片段，然后把 query 和这些片段拼在一起送给大模型生成答案。这样模型不需要记住所有知识，而是临时“查资料”。

和微调相比，RAG 的优势在于：第一，知识更新快，只需要更新知识库，不需要重新训练模型；第二，可以溯源，模型回答时可以引用检索到的文档，可解释性好；第三，适合处理长尾或动态的知识，比如实时新闻或企业内部文档。

劣势是：第一，推理时多了一个检索步骤，延迟更高；第二，检索质量直接影响最终效果，如果检索到不相关或错误的信息，模型可能被误导；第三，对模型的长上下文能力有要求，因为要塞入检索到的多段文本。

实际中两者常常结合：用微调让模型学会如何利用检索到的信息，再用 RAG 提供外部知识。
