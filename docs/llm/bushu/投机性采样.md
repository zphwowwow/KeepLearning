# **介绍一种投机性采样（Speculative Decoding）的方法。它为什么能加速推理而不改变输出分布？**

**一种典型方法**：用一个快的小模型（draft model）先跑几步，生成一串候选 token。然后用大模型（target model）一次性并行验证这些 token，把不合适的 reject 掉，只保留正确的前缀。这样一次大模型前向就能确认多个 token，而不是一个一个生成。

**为什么能加速**：因为小模型跑几步很快，大模型验证一次能确认多个 token。虽然验证那一步还是 O(n²) 的复杂度，但一次验证多个 token 摊薄了开销，整体吞吐就上来了。

**为什么不改变输出分布**：大模型做验证的时候，用的还是它自己的概率分布，只会接受那些概率大于等于小模型建议的 token。如果某个 token 在小模型那里概率很高，但大模型认为不行，就会被拒绝，然后回退到大模型自己生成。整个过程等价于大模型自己一步步采样，只是中间用了一个并行的方式来加速。所以输出分布和原始大模型完全一致，没有精度损失。
