# **量化方法：GPTQ、AWQ、GGUF的区别与选型**
这三种都是模型量化方法。GPTQ 是通过逐层的二阶近似来最小化量化误差，在 GPU 上推理速度快，精度损失小，但对激活值里的离群点比较敏感。AWQ 是根据激活值的分布，找出那些最重要的 1% 权重，不对它们做量化，所以处理离群点比 GPTQ 更好，精度也略高一些，但需要一个小的校准集。GGUF 是 llama.cpp 里用的格式，它支持混合精度，还可以把模型拆到 CPU 和 GPU 上混合推理，特别适合在只有 CPU 的边缘设备上跑，但速度肯定比纯 GPU 方案慢。

选型上，如果你有 GPU 且追求最高精度，选 AWQ；如果显存紧张但想快，选 GPTQ 的 4bit；如果只能在 CPU 上跑，比如树莓派或者手机，那就用 GGUF。

# 如果要将一个 13B 模型部署到单张 24GB 显存的 GPU 上（例如 RTX 4090），你会采用哪些量化方案？INT8、INT4、GPTQ、AWQ 各有什么优缺点？

**先算账**：13B 模型 FP16 大概 26GB，超过 24GB，所以 FP16 放不下。INT8 大概 13GB，INT4 大概 6.5GB，都能装下。但 INT8 质量好，显存余量小；INT4 显存充裕，可加 KV Cache 和 batch。

**INT8**：优点是无脑好用，精度损失极小，几乎无损。缺点是显存占用还是比 INT4 大一倍，且需要逐层做量化校准（比如使用 `llm.int8()`）。如果显存紧张，可能 batch size 受限。

**INT4**：优点是显存极低，能装下还能跑大 batch。缺点是精度损失明显，尤其对 small model（13B 算中等）可能掉点较多，需要额外技巧（如混合精度）。

**GPTQ**：优点是基于二阶近似的 4bit 量化，精度比普通 INT4 好很多，推理速度快。缺点是需要校准集，且对某些模型结构（如 MoE）支持不够好。

**AWQ**：优点是根据激活分布保护 1% 重要权重不量化，精度通常比 GPTQ 略好，尤其适合处理离群点。缺点同样需要校准集，实现比 GPTQ 复杂一点。

**选哪个**：如果追求最高精度，用 AWQ-4bit；如果追求速度且精度要求不高，用 GPTQ-4bit；如果不想折腾校准集，直接上 INT8（但 13B 用 INT8 勉强能跑，batch 只能为 1）。实际 4090 上，推荐 AWQ-4bit，显存占用约 7-8GB，剩余显存可以放 KV Cache 和跑 4-8 的 batch，比较实用。
