# Flash Attention 的原理和好处
Flash Attention 是一种在不牺牲精度的前提下加速 Attention 计算并减少显存访问的方法。传统 Attention 需要把 QK^T 这个 n×n 的矩阵完整写到 HBM（高带宽显存）里，这很慢而且显存占用大。Flash Attention 的核心技巧是分块计算和重计算。

它把 Q、K、V 切分成小块，在 SRAM（片上缓存）里完成一个块的 attention 计算，然后只把中间结果写回 HBM。为了不丢失信息，它用了一个 online softmax 的技巧，可以增量地更新归一化因子。另外，反向传播时它不保存完整的注意力矩阵，而是重新计算一部分，因为 SRAM 计算比 HBM 读写快。

好处：一是速度快，比标准实现快 2-4 倍；二是显存占用从 O(n²) 降到 O(n)，可以处理更长的序列。现在 Flash Attention v2 和 v3 已经普及，几乎成了大模型训练的标配。

# 没有 Flash Attention
在 Flash Attention 出现之前，标准 Transformer 的注意力机制实现存在明显的性能瓶颈，尤其当序列长度变长时，显存和计算开销会爆炸式增长。

## 标准注意力机制的计算过程
给定 Query、Key、Value 三个矩阵（尺寸均为 N × d，N 为序列长度，d 为特征维度），标准注意力通常按以下步骤计算：

1. **计算相似度矩阵**：`S = Q @ K^T`，尺寸 N × N。
    
2. **缩放**：`S_scaled = S / sqrt(d)`。
    
3. **行级 Softmax**：对 S_scaled 的每一行计算 Softmax，得到注意力权重矩阵 P（N × N）。
    
4. **加权求和**：`O = P @ V`。

这些操作在 PyTorch、TensorFlow 等框架中通常以 **独立的算子** 执行，每一步都会产生中间张量并写回到 GPU 的 **高带宽内存（HBM）** 中。

## 带来的三大问题

### 1. 显存占用为 O(N²)，随序列长度平方增长

- **核心瓶颈**：N × N 的注意力矩阵 S 和 P 都需要显式存储。
    
- 例如，序列长度 N=4096 时，一个 FP16 的注意力矩阵约 **128 MB**；  
    当 N=64K 时，单个矩阵就超过 **16 GB**，远超单张 GPU 的显存容量。
    
- 这直接限制了大模型处理长上下文的能力（当时主流模型上下文仅 512～2048）。
    

### 2. HBM 访问次数巨大，导致 IO 瓶颈

- 即使显存勉强装得下，标准实现需要反复从 HBM 中读写中间结果：  
    `Q @ K^T` → 写 S 到 HBM → 读 S → 缩放 → 写回 HBM → 读 S_scaled → Softmax → 写 P 到 HBM → 读 P → 读 V → 写 O。
    
- 对 HBM 的访问次数是 O(N²) 量级，而 HBM 带宽远低于 SRAM 带宽（约 1.5 TB/s vs 19 TB/s）。  
    结果就是 **计算单元大量时间空闲，等待数据搬运**，GPU 算力利用率极低。
    

### 3. 无法利用 GPU 的 SRAM 进行融合

- SRAM 速度极快但容量很小（如 A100 上 192KB）。标准算子设计为独立内核，每个内核只能看到自己的输入输出，无法将整个注意力计算链放入 SRAM 中。
    
- 因此即使知道 N² 矩阵巨大，也不得不把它写回 HBM。

#  **请推导 FlashAttention 的原理，它是如何在不损失精度的前提下减少 HBM 访问次数的？**
FlashAttention 的核心问题意识是：标准 Attention 在 GPU 上慢不是因为计算量大，而是因为 HBM（高带宽显存）读写太频繁。标准实现会把 QK^T 这个 n×n 的矩阵完整写回 HBM，再读出来做 softmax，再写回，再读出来乘 V。每一步都要跟慢速的 HBM 打交道，而 GPU 的 SRAM（片上缓存）速度极快但容量很小。FlashAttention 的做法就是：不让中间结果出 SRAM，分块计算，同时用数学技巧保证精度不丢。

具体推导是这样：把 Q、K、V 切成小块，让每一块能完整放进 SRAM。对于每一块，我们计算局部的注意力分数，但问题在于 softmax 是全局的——它需要所有 token 的最大值和分母求和。FlashAttention 用了一个 online softmax 的技巧：逐块累加，同时维护两个统计量，一个是当前见过的最大值，一个是当前的分母和。每处理一个新块，就用新的最大值去修正之前已经算过的 softmax 结果。这样最终算出来的结果和全局 softmax 完全一样，没有任何近似。

同时，反向传播时 FlashAttention 不保存中间的注意力矩阵（那个 n×n 的大矩阵），而是只保存每块的统计量（最大值和分母和）。反向时再重新计算一次注意力分数，虽然多算了一遍前向，但因为 SRAM 计算比 HBM 读写快得多，整体反而更快。

所以总结：FlashAttention 通过分块 + online softmax，让大部分数据在 SRAM 内处理完，只有最终结果写回 HBM。避免了把 n×n 的中间矩阵落地，HBM 访问次数从 O(n²) 降到 O(n)。而且所有操作都是精确的，没有精度损失。这也是它能成为大模型训练标配的原因。
