# 大模型推理加速技术有哪些？
可以分几个层面说：

第一是算子优化，比如 Flash Attention 和它的各种变体，减少显存读写；还有融合算子，把多个小操作合并成一个 kernel，减少 kernel launch 开销。

第二是量化，把模型权重从 FP16 压缩到 INT4 或 INT8，常用方法有 GPTQ、AWQ、GGUF。显存减半到减到 1/4，速度也相应提升。

第三是KV Cache 的优化，比如前面提到的 GQA 和 MQA，还有 PagedAttention（用在 vLLM 里），把 KV Cache 分页管理，减少显存碎片，提高批处理效率。

第四是解码策略的加速，比如投机解码（speculative decoding）：用一个快的小模型先 draft 出几个 token，再用大模型并行验证，一次生成多个 token；还有并行解码，比如 Medusa 在模型头部加多个解码头，一次性预测多个未来 token。

第五是服务框架层面的，如 vLLM 的连续批处理（continuous batching），不用等整个 batch 都生成完，而是随时插入新请求，提高 GPU 利用率。

实际部署中，通常会组合使用这些技术：比如 vLLM + AWQ + Flash Attention + 投机解码。

# 混合精度训练是什么？有什么注意事项？
混合精度训练就是在训练时同时使用 FP16（或 BF16）和 FP32。主要思路是：前向和梯度计算用 FP16，这样可以减少显存占用并加速计算；但梯度更新时用 FP32 的主权重，保证精度不丢失。

注意事项有几个：第一，FP16 的动态范围比较窄，容易下溢或上溢，所以需要损失缩放（loss scaling），先把 loss 乘一个大数，反向后再除回来。第二，现在很多卡支持 BF16，它的动态范围和 FP32 一样，只是精度低一些，通常比 FP16 更稳定，不需要 loss scaling。第三，某些操作比如 softmax 和归一化层，在 FP16 下可能不稳定，建议保持 FP32。第四，要小心梯度爆炸，FP16 下梯度一大就直接变 inf 了。

实际经验：能上 BF16 就上 BF16，省心很多。
